### SCR-00 대시보드

**목적:** 전체 프로젝트의 이해관계자·인터뷰·액션아이템 현황을 한눈에 파악

**SRS 추적:** MOD02-F-06, MOD02-NF-04

#### 화면 레이아웃

```
┌──────────────────────────────────────────────────────────────┐
│  📊 대시보드                                          [새로고침]│
├──────────────────────────────────────────────────────────────┤
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐        │
│  │ 이해관계자    │  │ 인터뷰 완료율 │  │ 미완료       │        │
│  │    👥 3명    │  │    📅 33%    │  │ 액션아이템   │        │
│  │              │  │  (1/3 완료)  │  │   ⚡ 2건     │        │
│  └──────────────┘  └──────────────┘  └──────────────┘        │
├──────────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────┐  ┌─────────────────────────┐│
│  │ 최근 인터뷰 계획             │  │ 미완료 액션아이템        ││
│  │ • [완료] 홍길동 - 2026-05-07│  │ □ 요구사항 초안 제출     ││
│  │ • [예정] 김영희 - 2026-05-10│  │   담당: 박개발 | 5/10   ││
│  │ • [예정] 이철수 - 2026-05-12│  │ □ UI 목업 공유          ││
│  └─────────────────────────────┘  └─────────────────────────┘│
└──────────────────────────────────────────────────────────────┘
```

**주요 UI 요소:**
- KPI 카드 3개: 이해관계자 수 / 인터뷰 완료율 / 미완료 액션아이템 수
- 최근 인터뷰 계획 목록 (상태 뱃지: 예정/완료/취소)
- 미완료 액션아이템 목록 (담당자, 마감일 표시)
- 각 항목 클릭 시 해당 화면으로 이동

**사용자 흐름:** 앱 진입 → 대시보드 → KPI 클릭/항목 클릭 → 해당 상세 화면

### SCR-01 요구사항 관리

**목적:** RFP 요구사항 JSON을 불러와 조회하고 인터뷰 대상을 선택

**SRS 추적:** MOD02-F-01-01, MOD02-F-01-02, MOD02-F-01-03

#### 화면 레이아웃

```
┌──────────────────────────────────────────────────────────────┐
│  📋 요구사항 관리                                             │
│  [📂 JSON 불러오기]                  검색: [___________] 🔍   │
├───────┬──────────────────┬──────┬──────────┬──────┬──────────┤
│  선택  │ 요구사항 ID       │ 수준 │ 요구사항명│ 우선순위│ 상태   │
├───────┼──────────────────┼──────┼──────────┼──────┼──────────┤
│  ☐   │ REQ-001           │  1  │ 이해관계자 | Must  │ Active  │
│  ☑   │ REQ-002           │  2  │ 인터뷰 계획│ Must  │ Active  │
│  ☐   │ REQ-003           │  2  │ 질문지 관리│ Should│ Active  │
│  ...  │ ...               │ ... │ ...       │ ...   │ ...     │
├───────┴──────────────────┴──────┴──────────┴──────┴──────────┤
│  선택된 요구사항: 1개    [인터뷰 계획에 연결 →]                │
└──────────────────────────────────────────────────────────────┘
```

**주요 UI 요소:**
- [JSON 불러오기] 버튼: 파일 선택 다이얼로그 트리거
- 요구사항 그리드: 체크박스 선택, 컬럼 정렬/검색
- 선택 개수 표시 + [인터뷰 계획에 연결] 버튼
- 행 클릭 시 상세 패널(우측 슬라이드) 표시

**오류 처리:** 잘못된 JSON → 빨간 토스트 메시지 표시

- [버전 드롭다운]: 이전 버전 불러오기
- [저장]: 현재 버전 저장

**사용자 흐름:**
1. SCR-03 인터뷰 목록 → [질문▶] 클릭
2. SCR-04 진입 → [AI 질문지 생성] 클릭
3. 로딩 후 우측 제안 목록 표시
4. 항목별 수락/거부 처리
5. 확정 질문 순서 조정
6. [저장] 클릭 → 버전 증가

- 4탭 구조 (원본 메모 → 구조화 → 매핑 → 액션아이템)
- 탭 1: 텍스트 에디터, [저장], [AI 결과 정리] 버튼
- 탭 2: 구조화 항목 카드, 수락/거부 버튼, 액션아이템 후보 배지(⚡)
- 탭 3: 매핑 테이블, 추론 배지, [매핑 실행] 버튼
- 탭 4: 액션아이템 등록 폼 (담당자, 마감일)

### SCR-CM 공통 컴포넌트

**SRS 추적:** MOD02-NF-04 (RETS 디자인 시스템 준수)

#### CM-01 토스트 메시지

| 유형 | 색상 | 사용 상황 |
|------|------|-----------|
| 성공 (Success) | 초록 | 저장 완료, 파일 다운로드 완료 |
| 오류 (Error) | 빨강 | JSON 파싱 실패, LLM 호출 오류 |
| 경고 (Warning) | 주황 | 필수값 누락, 삭제 확인 요청 |
| 정보 (Info) | 파랑 | LLM 처리 시작 안내 |

#### CM-02 로딩 인디케이터

- LLM 호출 시 버튼이 비활성화되고 스피너 아이콘으로 전환
- "AI가 분석 중입니다…" 텍스트 표시
- 응답 수신 후 즉시 결과 패널 표시 및 버튼 복구

#### CM-03 확인 다이얼로그

- 삭제 작업 전 "정말 삭제하시겠습니까?" 모달 표시
- [확인] / [취소] 버튼 제공

#### CM-04 언어 전환 (한국어/영어)

- 푸터의 [KO/EN] 버튼으로 UI 전체 언어 전환
- 선택값은 localStorage에 저장하여 재접속 시 유지
