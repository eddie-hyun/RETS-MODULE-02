## 1. 테스트 설계 개요

### 1.1 목적

본 문서는 RETS-MODULE-02의 SRS(MD03)에 정의된 기능 요구사항을 검증하기 위한 테스트 시나리오와 테스트 케이스를 정의한다. TDD(Test-Driven Development) 원칙에 따라 구현 전 테스트를 선행 설계한다.

### 1.2 테스트 범위

| 시나리오 그룹 | SRS 요구사항 | 우선순위 |
|---------------|-------------|---------|
| TS-01 요구사항 목록 로드 | MOD02-F-01-01 ~ 03 | Must |
| TS-02 이해관계자 관리 | MOD02-F-02-01 ~ 03 | Must |
| TS-03 인터뷰 계획 관리 | MOD02-F-03-01 ~ 03 | Must/Should |
| TS-04 질문지 관리 | MOD02-F-04-01 ~ 04 | Must/Could |
| TS-05 인터뷰 결과 처리 | MOD02-F-05-01 ~ 06 | Must/Should |
| TS-06 대시보드 | MOD02-F-06 | Must |

### 1.3 테스트 유형

- **기능 테스트(FT)**: 요구사항 준수 여부 확인
- **경계값 테스트(BV)**: 입력 경계 조건 확인
- **예외 처리 테스트(EX)**: 오류 상황 대응 확인
- **UI 테스트(UI)**: 화면 표시 및 인터랙션 확인
