## 1. 프로젝트 개요

### 1.1 사업 배경

본 프로젝트는 사내 요구공학 도구 스위트(RETS; Requirements Engineering Tool Suite) 구축 사업의 일환으로, **RETS-MODULE-02 — 이해관계자 및 인터뷰 매니징** 모듈을 개발하는 PoC(Proof of Concept) 프로젝트이다.

요구공학 초기 단계에서 이해관계자 식별, 인터뷰 계획 수립, 질문지 관리, 인터뷰 결과 기록·정리, 요구사항 매핑 등의 업무가 비체계적·수동적으로 수행됨으로써 인터뷰 품질 편차와 정보 파편화 문제가 발생하고 있다. 본 모듈은 이러한 문제를 해결하기 위한 AI 보조 기반의 웹 도구이다.

### 1.2 프로젝트 목표

| 구분 | 내용 |
|------|------|
| 개발 대상 | RETS-MODULE-02: 이해관계자 및 인터뷰 매니징 웹앱 (Vanilla HTML) |
| 개발 방식 | 단일 HTML 파일 기반 프론트엔드 PoC (백엔드 없음) |
| LLM 활용 | LLM 릴레이 서버 (Google Gemini 기반) 연동 |
| 최종 산출물 | 실행 가능한 웹앱 + 전체 문서 산출물 (MD01~MD11) |
| PoC 목표 | 핵심 유저 스토리 5개 이상 구현 및 시연 |

### 1.3 개발 범위 요약

- 이해관계자 등록·관리 및 영향력(Power/Interest) 매트릭스 시각화
- 인터뷰 계획 수립 및 일정 관리 (캘린더/목록 뷰)
- LLM 기반 인터뷰 질문지 자동 생성 및 편집
- 인터뷰 결과 기록, 구조화 정리, 요구사항 매핑, 액션아이템 도출 (LLM 보조)
- 인터뷰 수행 결과 파일 출력 (Markdown / CSV)

## 2. 팀 구성 및 역할 분담

### 2.1 팀 구성

| 역할 | 담당 업무 |
|------|-----------|
| **PM (프로젝트 매니저)** | 전체 일정 관리, 산출물 품질 검토, 리스크 모니터링, PoC 발표 준비 |
| **기획/요구분석 담당** | RFP 분석, PRD·SRS 작성, 유저스토리 정의, 벤치마크 시나리오 설계 |

| **프론트엔드 개발 담당 (1)** | HTML 앱 코어 기능 구현 (이해관계자 관리, 인터뷰 계획) |

| **QA / 문서 담당** | 테스트 설계(TDD), 테스트 수행(TR), 사용자 가이드(UG) 작성 |
